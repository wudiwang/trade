# 形态→策略 闭环 & 工作台设计(用户 2026-06-16)

## 你想要的闭环
贴一张"漂亮的买点图" → 出策略 → 写脚本回测 → 把合格点位以图发回甄别 → 你给反馈 → 改 → 直到OK → 上模拟盘。

## 关键判断:不需要"传图给Agent"的独立页面
- **看图理解 + 写策略代码,必须由带视觉的 Claude(主会话)完成** —— 也就是你现在贴图的这个聊天。
- 子 Agent(Agent工具派生)**收不到你贴的图片**(只接收文字prompt),所以"图→策略"这一步天然属于主会话,不该塞进子Agent或自建上传页。
- 自建一个"上传图片→后台调LLM"的页面 = 重复造聊天,得不偿失。

## 真正需要的是「策略工作台」页面(= 看图器升级版)
它是**人类驾驶舱**,不是Agent入口。三件事:
1. **总览**:每个策略当前漏斗阶段(草案/回测中/ITERATE/待审美/PROMOTE/已否决)+ 最新成绩单(扣费净期望)。
2. **甄别**:某策略的合格点位列表,逐张看图,**👍/👎 + 一句评论** → 存进 `docs/agents/feedback_<strat>.jsonl`。
3. **跳转**:点开即现有看图器的K线+标记。

## 各步由谁做(诚实分工)
| 步 | 动作 | 谁 | 产物 |
|---|---|---|---|
| 1 | 看图 → 出策略草案 | **主会话Claude(你贴图)** | `strat_<name>.py` + 规格doc + 注册表条目 + notes |
| 2 | 写脚本/回测/扣费/扫参/优化 | 脚本 + strategy-researcher子Agent | 成绩单JSON + notes更新 |
| 3 | 合格点位以图发回 | 工作台(researcher精选哪些) | 甄别清单 |
| 4 | 你甄别 👍/👎/评论 | **你** + 工作台 | `feedback_<strat>.jsonl` |
| 5 | 据反馈改逻辑/参数 → 重跑 | 主会话或researcher | 新版策略 + notes |
| 6 | 循环2~5 直到你OK | — | 定稿 |
| 7 | 上模拟盘前向验证 | 脚本(paper_tracker) | 样本外成绩 |

## "子Agent帮我跟进"到底是什么形态
- 子Agent **不长跑**(LLM长跑又贵又没必要)。每个策略的"身份与记忆" = 它的 `notes_<strat>.md` + `feedback_<strat>.jsonl` 文件。
- "跟进" = 每当①有新数据 或 ②你给了新反馈 时,**唤醒一次**该策略的 researcher,它读notes+feedback,接着上一轮继续。
- 持续性(每日补数据、夜间出成绩单)由**脚本+计划任务**扛,不耗LLM。

## 落地顺序建议
1. **工作台甄别+反馈**(看图器加 👍/👎/评论 → 写 feedback 文件):闭环里"人给反馈"这一环,最关键、最小改动。
2. **状态总览面板**:把各策略成绩单+漏斗阶段汇总到工作台首页。
3. **图→策略 的主会话规范**:贴图时我按固定模板产出"草案策略+规格+注册",并自动建 notes/feedback 文件。
4. paper_tracker(样本外前向),最后再做。

## 一句话
**聊天=出策略的引擎(看图+写码);工作台=你审美甄别+下反馈的驾驶舱;脚本+计划任务=常开的肌肉;子Agent=按轮唤醒的研究员。** 不需要再造一个"管理Agent"的独立后台。
