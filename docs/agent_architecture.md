# 多策略·多Agent 研究架构(用户 2026-06-16)

目标:**用最少的时间,把"市面上的 + 我自己的"各路策略,逐个验证到一个明确结论**(淘汰 / 继续调 / 推进模拟盘),并让我能快速肉眼复核图形是否符合审美。

---

## 核心原则:确定性的归脚本,判断性的归 Agent
> 回测 627 币只要 10 秒,是确定性计算 —— 绝不让 LLM 去"算"。
> LLM Agent 只做**判断**:边际真不真、怎么调、图形合不合审美、是否过拟合。

这样 LLM 成本可控、可并行、可一直挂在本地。

---

## 三层架构

```
┌─ 第0层 · 确定性引擎(本地脚本, 一直开, 不花钱) ───────────────┐
│  • 每日增量拉数据      bt_refresh.py  (Task Scheduler 定时)      │
│  • 每策略夜间回测+扫参  bt_scorecard.py → 写 scorecard_<strat>.json │
│  • 前向/模拟盘跟踪      paper_tracker  (新数据上的样本外结果)      │
│  • 可视化看图器        bt_viewer.py   (你肉眼复核)               │
└──────────────────────────────────────────────────────────────┘
            │ 产出: 每个策略一份"成绩单"JSON + 代表性样本
            ▼
┌─ 第1层 · 策略研究 Agent(每策略一个, 定时或按需唤醒) ─────────┐
│  读成绩单 → 判断边际/稳健性/过拟合 → 提调参建议 →               │
│  精选 6~10 个代表样本(典型盈/典型亏/边界)交你审美 →           │
│  给结论: 淘汰 / 继续调(具体改什么) / 推进模拟盘               │
│  各自写笔记 docs/agents/notes_<strat>.md(独立记忆)            │
└──────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第2层 · 编排 Agent(1个, 总指挥) ───────────────────────────┐
│  汇总所有成绩单 → 排行榜 → 决定把人力/算力投给谁、淘汰谁 →      │
│  避免重复劳动 → 报告"当前最优 + 下一步"                        │
└──────────────────────────────────────────────────────────────┘
            │
            ▼
        你(+ 看图器):审美复核 → 拍板推进模拟盘
```

## "最少时间"漏斗(快速淘汰是最大的省时)

| 闸 | 谁 | 判据 | 动作 |
|---|---|---|---|
| **A 成本闸** | 脚本(夜间) | 扣手续费后,1周/2周/1月三档期望 | 全负 → **自动归档为"否决"**,不再浪费判断 |
| **B 稳健闸** | 策略Agent | 跨档/跨方向一致?是否被个别离群单撑起? | 不稳 → 提具体调参 or 淘汰 |
| **C 审美闸** | 你+看图器 | Agent 精选样本,图形是否符合你心中的形态 | 不像 → 退回改逻辑 |
| **D 前向闸** | 脚本(持续) | 样本外(最近新数据)期望是否保持 | 过拟合则降级 |

> 关键:**A 闸自动跑、自动毙**。你和 Agent 的时间只花在过了 A 闸的少数幸存者上。

## 关于"独立 Agent"的优化建议
不必为每个策略手写一份几乎相同的提示词(4份会各自漂移、难维护)。
**用同一个"策略研究员"角色提示词,每次实例化锁定一个策略**(靠 `--strat` 参数 + 各自的 notes 文件保持独立身份与记忆)。
- 好处:DRY、加新策略零成本(不用写新提示词)、行为一致可对比。
- 仍然"独立":每个实例只看自己那一个策略、写自己的笔记、互不干扰。
- 提示词模板见 `docs/agents/strategy_researcher.md`;总指挥见 `docs/agents/orchestrator.md`。

## 执行方式(本地常开)
- **第0层**:Windows 计划任务 —— 每日刷新数据 + 夜间生成所有策略成绩单。**免费、无人值守。**
- **第1层**:策略研究 Agent —— 每天数据更新后唤醒一次(或你按需触发),读成绩单做判断。**LLM 成本 = 每策略每天一次轻量分析。**
- **第2层**:编排 Agent —— 每天一次,出排行榜与下一步。

LLM 不"长跑",只在"有新数据/你召唤"时做判断 —— 这是成本与效果的最优点。
