# 项目玩法说明书(如果以后忘了怎么玩,先读这篇)

> 一句话:这是一套**个人加密合约交易系统**,分两半——
> **本地研究实验室**(你这台常开的Windows,做策略/回测/看图/打标)
> + **VPS实盘执行室**(出信号→小飞机推送→你确认→真实下单)。
> 原则:**本地随便试,实盘版本化、可审计、你点头才动。**

## 一、整体架构(两个应用,一个仓库)
```
本地研究实验室(这台机器)                VPS 实盘执行室(76.13.182.175)
  bt_viewer 看图器 :8530                 trade.service(systemd)
  策略 strat_*.py + 回测 + 打标            macro_pullback 实盘策略
  多Agent研究 + 审美反馈闭环               Web面板 :8488 + Telegram机器人
  .btcache 本地数据(628币×3级别×30天)     币安USDT永续 真实下单(paper/live)
            └── 验证通过的策略 → 部署到VPS ──┘
```

## 二、实盘是怎么跑的(真金路径)
1. VPS引擎逐根评估K线 → 出信号(当前实盘策略=`macro_pullback`)
2. 信号推到**Telegram小飞机**,卡片含:入场/止损/止盈、**名义额/保证金/杠杆、预期盈利/亏损、当前持仓、可用保证金**、图形链接
3. 你点 **[✅确认买入]** → **live模式下=真实下单**(市价进场 + 自动挂止损/止盈)
4. `auto_trade=关` 时才要你手动确认;开了=免确认直接下单(有熔断/持仓上限)

### 💰 每笔下单金额怎么算(优先级,谁先>0谁说了算)
`① 单笔保证金U → ② 单笔保证金% → ③ 单笔名义额U → ④ 风险算法`
当前:②保证金%=10 → 本金100U×10%=**10U保证金 × 5倍杠杆 = 50U名义额**。
这些参数在 **Web面板 :8488 → 参数配置 → 🔴线上配置项** 分组里改,点"保存并热生效"立即生效。

### 🔴 线上配置项(直接影响真金,改前想清楚)
运行模式(paper/live) · 自动下单 · 账户本金 · 杠杆 · 单笔保证金%/保证金U/名义额U · 实盘最大持仓 · 亏损熔断%

## 三、本地研究怎么玩
- **数据**:`scripts/bt_refresh.py` 每天增量补5m/15m/1h(计划任务 `TradeBacktestRefresh` 每天08:30自动跑 daily_job.cmd:补数据→重算信号)
- **策略**:每个独立成 `app/engine/strat_*.py`,注册进 `scripts/bt_registry.py`
- **回测**:`scripts/bt_scan.py` 预算信号→`.btcache/sig_*.json`;`bt_registry.score()` 出扣费后期望
- **看图器**:`.venv/Scripts/python scripts/bt_viewer.py` → 浏览器 **http://127.0.0.1:8530**
  - 信号表(按策略/方向/结果筛)→ 点开看K线 + 入场/止损/止盈线 + 形态标记
  - 5m/15m/1h 级别切换、策略详情面板、数据新鲜度、⭐保存案例
  - **👍/👎 审美打标 + 理由**(写进 `pattern_cases/labels.jsonl`)
  - Agent工作台 `/agents`

## 四、多Agent + 审美反馈闭环(核心玩法)
**理念:确定性归脚本(回测10秒),判断归Agent;你的眼睛=最终标准。**
```
① 打标: 看图器对信号标 👍符合/👎不符合 + 理由(理由=以后收紧公式的依据)
② 每天定时: 分析脚本对比 👍vs👎 特征 → 【先出一份报告】(不改公式!)
③ 你看报告 → 确认 → 才改公式 → 记审计日志(基于N张图改了什么、通过率前后)
④ 整体通过率↑ = 成功
```
- 子Agent:`.claude/agents/strategy-researcher`(研究单策略到结论)、`strategy-orchestrator`(排行榜)
- 调用:跟Claude说「用 strategy-researcher 研究 STRAT=xxx」
- 每个策略的记忆 = `docs/agents/notes_<策略>.md`

## 五、运维操作(常用命令)
| 要做的事 | 命令 |
|---|---|
| 部署到VPS | `git archive <commit>` → scp → tar解包 → 写REVISION → `systemctl restart trade` |
| 只读核验线上 | `deploy/verify_remote.ps1`(查版本/服务/web/关键函数) |

## 六、安全铁律
1. `.env`/密钥**绝不**碰、不打印
2. 实盘部署前**核验版本+健康**(verify_remote);**未经你确认不开 auto_trade**
3. 改公式前**先出报告 → 你确认**;每次改留审计日志
4. **不删** `.btcache/` `data/` `logs/`;不在实盘代码上做实验
5. 并发的 codex 会话也会改这个仓库 → **部署前确认版本对**(曾因部署错版本把信号掐了8.5h)

## 七、出问题怎么查(速查)
- **小飞机没信号** → 查:服务active? K线在写入?(说明引擎在评估)journal有无异常?是不是版本被换了?是不是市场安静(信号事件驱动,几小时空档正常)
- **网页不自动刷新** → 缺 `websockets` 库;`pip install 'uvicorn[standard]'` + 重启
- **服务崩溃循环** → `journalctl` 看 Traceback;通常是某处对币安返回数据没容错
