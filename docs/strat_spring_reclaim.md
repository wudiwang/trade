# 策略规格 ·「深跌 → 弹簧 → 起跌位收回 → 平台企稳」二段式做多

> 状态:**设计稿,仅供回测验证,未接实盘、未改引擎**(遵循策略独立打包存档流程)。
> 来源:用户 2026-06-16 多张图分析逐步细化(深跌后企稳 + 威科夫弹簧 + 起跌位支撑 + 二段建仓)。

---

## 一、一句话定义
在**大级别深跌的底部区域**,出现一根**爆量下跌标志K**插穿前低(诱空),随后价格**收回到这根标志K的"起跌位"(开盘价)之上**,并在该位置**横盘得到支撑**;支撑验证有效后,先**轻仓**介入,平台内再出现**底分型**则**加仓**。只做多。

---

## 二、三层时间框架分工

| 层 | 级别 | 职责 |
|---|---|---|
| **背景层** | 1h | 确认"跌了很多 + 处于低位 + 趋势向下"(deepbase A层),只在底部区域才允许触发 |
| **结构层** | 5m | 找爆量下跌标志K、判定收回、横盘企稳、平台底分型 |
| **定位层** | 1m | 精确定位标志K的**起跌位(开盘价)**与**针尖(最低点)**;精确捕捉"收回起跌位"的瞬间 → 更紧的止损、更早的入场 |

核心思想:**大级别定方向/区域,5m定结构,1m定扳机与止损。**

---

## 三、关键价位(全部围绕"爆量下跌标志K"展开)

设爆量下跌标志K为 **C**:
- **P_start = C 的开盘价** = "起跌位" = 核心支撑参考位 / 收回目标位
- **P_spring = C 的最低点(1m 精修针尖)** = 弹簧极值 = 止损基准
- 插穿幅度 = (P_start − P_spring) / P_start

---

## 四、信号流水线(逐阶段,带量化条件)

### 阶段0 · 背景闸(1h,必过)
- 近 `w1`(默认48)根1h:(最高−最低)/最高 ≥ `drop_min`(默认30%)
- 现价处于该区间下部 `pos_max`(默认50%)以内
- 收盘 < 1h MA(`ma_1h`=50) 且 该均线斜率向下
- **不过 → 直接放弃**(避免在上涨/震荡市抓"弹簧"产生大量假信号)

### 阶段1 · 识别爆量下跌标志K(C)
- 在近 `cap_lookback`(默认30)根5m内,找满足以下全部的K:
  1. **下跌K**:close < open
  2. **爆量**:成交量 ≥ 前`vol_ma`(20)根均量 × `climax_mult`(默认3.0)
  3. **做新低/插穿**:其最低 = 窗口内最低(或刺穿了前一企稳平台/前低)
- 命中后用 **1m 精修**:在 C 的时间区间内取 1m 最低 = P_spring(针尖);P_start = C 开盘价
- 多根候选时取**量最大且做最低**的那根

### 阶段2 · 收回起跌位(弹簧成立)
- C 之后 `reclaim_bars`(默认6,按定位级别计)内,价格**收盘收回到 P_start 之上**(用1m判定"回到起跌位的瞬间")
- 收回必须是**收盘价**站上 P_start,不是只有上影线碰一下(防假收回)
- 期间**未出现收盘 < P_spring**(针尖未被有效跌破)

### 阶段3 · 横盘得到支撑 → 轻仓(Entry-1)
- 收回后,连续 `hold_bars`(默认4)根满足:
  - 不再创新低(low 不破区间)
  - 不顺势下跌:收盘维持在 P_start 附近的带内(波幅 ≤ `hold_tol`,默认 P_start×3%)
  - **硬失效**:任意一根收盘 < P_spring → 整个 setup 作废
- 满足 → **支撑确认 → 轻仓买入(Entry-1)**
  - 入场 = 当前收盘;止损 = P_spring 下方一点(`sl_buf`);止盈 = 按 RR 或平台上沿

### 阶段4 · 平台底分型 → 加仓(Entry-2)
- 在企稳平台内(P_spring 之上)出现一个**底分型**(复用 `find_fractals`/`build_bi`)
- 底分型确认(右K收回)→ **加仓(Entry-2)**
  - 加仓止损可上移到底分型最低 或 仍用 P_spring

### 全程失效条件
- 任意阶段出现**收盘 < P_spring** → 弹簧失败,放弃/止损离场。

## 五、程序化监控可行性

### 回测(历史)
- 1h、5m:REST 直接拉,已有 `fetch_series`。
- 1m:**只在阶段1命中标志K的那几个时间窗附近补拉**(deepbase 命中很稀有,7天全市场约62次)→ 数据量极小,不会触发大面积限速。
- **结论:回测完全可行。**

### 实盘(实时)
- 现状:WS 只订 `5m/15m/1h`([config.yaml:18]),**未订 1m**。
- **推荐做法(避免12倍消息量):分级唤醒**
  1. 用 5m/15m/1h(现有WS)跑背景闸 + 标志K识别 → 把命中的币放进"弹簧观察池"
  2. **只对观察池里的少数币** REST 轮询 1m(每3–5秒)来定位收回/针尖/平台
  3. 出局或超时后移出观察池
- 这样 1m 只服务于"正在筑底的极少数币",带宽/CPU 几乎无增量,延迟(REST 1m)也够用。
- **结论:实盘可行,但需要新增"观察池 + 1m轮询"这一层(中等改动),且暂不接自动下单。**

## 六、需要特别注意的点(直接影响正确率)

| # | 风险 | 对策 |
|---|---|---|
| 1 | **标志K界定模糊**(多根大量K) | 量×做最低双条件,取唯一最强;必须是下跌K |
| 2 | **假收回**(上影线碰P_start) | 收回必须**收盘价**站上 P_start |
| 3 | **假弹簧/二次插针** | 硬失效:收盘 < P_spring 即作废,快速止损 |
| 4 | **横盘判定太松** | hold_bars + 波幅带 + 不创新低三重约束 |
| 5 | **前视偏差(回测致命)** | 每阶段只在"确认那根K"成立时入场,严禁用未来K |
| 6 | **1m/5m 对齐** | 标志K在5m定位,针尖/收回在该5m时窗内的1m精修 |
| 7 | **小币插针噪声** | 24h成交额下限过滤,低流动性直接排除 |
| 8 | **重复触发** | 按标志K(C)的 open_time 去重,一次筑底只一组信号 |
| 9 | **背景闸缺失会泛滥** | 1h深跌闸是第一道也是最重要的过滤 |
| 10 | **二段建仓的统计** | 回测要把 Entry-1/Entry-2 分开统计(分别看胜率/期望) |
