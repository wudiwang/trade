## 1. 用户的核心理念（纲领，所有策略的指导）
- **大盘第一**：下跌趋势中任何上涨都可能酝酿更强下跌（反弹=诱多）。做任何信号都要先看BTC大盘方向。
- **流动性买点**：最好的买点 = 价格"破掉"关键位(前高/前低/流动性所在) → **假突破再跌回收**(sweep + reclaim)。这正是V4策略的灵魂，要继续强化为最高优先级买点。
- **大势研判源**：YouTube博主「提阿非罗」每日(周一~周五)更新BTC大盘视频，从日线/周线/4h做方向研判，指出流动性所在和好的介入位。其观点作为重要参考/提醒点。
- **顺藤摸瓜**：BTC研判 → 顺势研究 ETH/SOL 大市值币 → 再到优质山寨。
- **操纵识别**：山寨币持仓数据(大户私有比例、筹码集中)是判断"是否被操纵/能否逼空暴涨"的关键。如 PEPE：链上显示大部分代币被大户私有、流通极少 → 合约价差大、费率贵 → 暴涨后继续暴涨。
- **多维消息面**：宏观消息 + 社群讨论热度 + KOL带货 作为关注/加分因子（讨论声高 + K线低位 → 进关注列表）。

## 2. 分阶段规划

### ✅ 可自主构建（loop 现在就能推进）

**P1 流动性核心策略强化**
- 显式识别并标注"流动性位"：前高/前低/密集成交区/等高的多个低点(等待被扫)。
- "扫流动性+假突破回收"(sweep & reclaim)作为最高优先级买点，独立标记、优先推送。
- V4已有雏形(假破并入二买)，这步把"流动性位"显式化并提级。

**P3 预演 / Playbook 功能（重点新功能）**
- 新DB表 `playbooks`：为某币记录一个剧本——当前形态描述、预判买点/卖点/目标位、触发条件(如"跌到X位"、"假突破Y后回收")、状态。
- 网页：新增/编辑/列表预演；点击看该币K线并标注预演位。
- 监控：引擎每根收盘K检查所有活跃预演，价格到达预演位或剧本成立 → 推TG提醒。
- 强阻力K线信号的山寨币 → 自动建档预演 + 买卖点预判 + 存档。

**P4 关注列表 Watchlist**
- 手动加币 + 理由(讨论声高/K线低位等)。关注列表内的币信号优先推送、单独成组展示。

**P5a 操纵/逼空检测（币安免费衍生数据）**
- 接入币安 fapi 衍生数据：持仓量OI(/fapi/v1/openInterest)、多空账户比、大户多空持仓比、资金费率、主动成交比。
- 组合识别"筹码集中+逼空"特征：OI骤增 + 费率极值 + 价格仍在低位 → PEPE式信号，标注"⚠逼空候选"。

**P2a 大盘研判覆盖层（手动输入版）**
- 网页一个"大盘观点"输入框：每天把提阿非罗对BTC的方向研判(如"下跌趋势,反弹诱多"/方向=空)录入。
- 系统把它作为**方向过滤/提醒覆盖**叠加到所有信号：逆大盘方向的信号打"⚠逆大盘"警告或自动降权。
- 多级别(日/周/4h)BTC自身结构分析作为辅助佐证。

### ⏳ 需用户提供数据源/密钥后再做

**P2b 大盘研判自动化** — 频道已定：https://www.youtube.com/@tiabtc 。方案：频道RSS取最新视频(无key) → 字幕(youtube-transcript-api,无key,但VPS数据中心IP可能被YouTube限流,需实测) → LLM摘要方向观点写入"大盘观点"字段。摘要走"每日Claude例程"(用我的能力,不用额外LLM付费key)而非VPS内置LLM。⚠风险：服务器IP抓字幕可能被封,备选=Claude例程在本地/云端跑。
