# 交易系统准则（DOCTRINE.md）—— 最底座

> 用户 2026-06-16 总结的交易系统**最底座**。核心：交易是**自上而下、逐层推导**的过程。
> 这是所有策略、因子、买点的**纲领**，优先级高于具体策略。任何信号/买点都必须能追溯到这套层级。
> 与其他文档关系：本文=为什么交易(决策框架)；ROADMAP.md=系统怎么演化；RESEARCH.md=用什么因子。

## 一、第一原则：决策层级不可跳级

做交易的顺序是固定的，**缺上层，则下层无效**：

```
① 大盘环境(趋势定义)  →  ② 预判(方向假设)  →  ③ 交易策略(计划/区间)  →  ④ 找买点  →  ⑤ 止盈止损
```

**硬规则：没有 ①② 这两步，就不要去做 ④ 找买点。** 脱离大盘环境和预判去找买点，是无意义的。
"先有预判，才有交易策略；有了预判和策略，才是找买点。"

---

## 二、第①层：大盘环境与趋势定义

- **大盘第一，永远先看 BTC。** 用「趋势转换信号」给 BTC 定义当前趋势。
- 趋势只有三种基调，先把当前归到其一：
  1. **上升趋势**
  2. **下降趋势**（任何上涨都可能是诱多反弹，酝酿更强下跌）
  3. **盘整/区间震荡**（方向不明朗）
- **趋势的改变有明确信号** → 据此建立「大盘策略信号联动关系」：先预判 BTC 会出现怎样的走势，用趋势转换判断来**重新定义** BTC 趋势，趋势变了，下游策略跟着切换。
- 定义好趋势后，**只顺趋势交易**。

## 三、第②层：预判（方向假设）

- 预判 = 基于结构证据、**可证伪**的方向假设。要有依据，不能拍脑袋。
- **示范案例（本准则的标准范式）**：
  - BTC 急跌到 **60,000**；
  - **68,000 存在真空缺口（FVG）** → 大概率会被回补；
  - 上周**弱反弹后，始终没跌破底部横盘的 POC** 那条位置 → 底部支撑有效；
  - **综合预判：BTC 大概率上探 68,000。**
- 预判用到的核心结构证据：**FVG（真空缺口）、POC（横盘密集成交）、关键支撑/阻力、反弹强弱**。（这些因子的研究见 RESEARCH.md）

## 四、第③层：交易策略 / 计划

有了预判，才能制定计划——**确定在什么区间范围内开始找买点**。

- 案例对应的两个方向：
  1. **追突破**：等 BTC 突破顶部时找机会；
  2. **守安全区**：在更安全的区域（如 68,000 附近）找**山寨币**买点。
- **方向不明朗 / 盘整震荡时**：先为未来走势**定下一个大基调**，再顺着这个基调去制定策略。
- 顺藤摸瓜：BTC 研判 → ETH/SOL 等大市值 → 优质山寨。

## 五、第④层：找买点

- **只在第③层框定的区间和条件内找买点**，工具是具体策略（缠论笔 chan_bi、流动性买点 sweep & reclaim 等）。
- 买点**必须顺趋势/顺基调**。逆基调的买点不做。

## 六、第⑤层：止盈止损

- 买入之后，再制定止盈、止损。风险管理是最后一层，不是第一层。

---

## 七、关键定义

- **顺趋势交易**：在**已定义的趋势方向内**、成功率较高的那类交易策略。不是预测，是站在概率高的一侧。
- **最底座**：以上 ①~⑤ 层级即交易系统的地基。**有了这个底座，再去选标的、观察、操作**——选标的是地基之上的事，不能反过来。

## 八、落地映射（系统怎么承接这套准则）

| 准则层级 | 系统模块 / 待办 |
|---|---|
| ① 大盘环境 | `macro_view`（大盘观点）、BTC 趋势定义与趋势转换信号、`btc_filter` |
| ② 预判 | FVG / POC / 结构识别（见 RESEARCH.md：Volume Profile、ICT/FVG） |
| ③ 策略/区间 | 预演 Playbook（到价/假突破回收触发）、区间框定 |
| ④ 买点 | `chan_bi` 笔买点 + 流动性买点(sweep & reclaim) |
| ⑤ 止盈止损 | 分型/流动性止损、Volume Profile 密集区止盈、RR 门槛 |

> 检查清单（每笔交易自查）：这笔买点，能不能一路往上追溯到「顺 BTC 趋势 + 有明确预判 + 落在计划区间内」？答不上来，就不该开仓。

## 九、系统底层思路：两段式漏斗（选标的 → 择时）

交易系统的执行架构，分两层，**先选标的、再选买卖点**：

```
Stage 1  选标的(screener)  →  从全市场动态筛出「符合特征」的候选池(不是固定币池)
            ↓  只在候选池上跑
Stage 2  择时(买卖点引擎)  →  chan_bi 等出信号，才触发交易
```

- **分层意义**：把「做什么」(selection / 找 alpha) 和「何时做」(timing / 执行) 解耦。两层各自可回测、可调参。
- 现状：已有半套 —— `universe`(按流动性选币) + `squeeze.py`(OI暴增+费率极值+低位 → 逼空**做多**候选)。需补的是**做空方向的 screener**。

### 9.1 定向机会：做空「涨得最凶」的币（见顶反转）
监控全市场涨幅/加速度最高的币，等**卖点**出现做空。收益可观（pump 后的 dump 又快又狠），但**最危险**，受以下铁律约束：

1. **逆势交易，必须用大盘 gate**：BTC 走弱/震荡时做空逆势硬拉的 alt = 顺大盘+反 alt 局部势 = 高胜率；**BTC 强多时此策略关闭或极小仓**（防逼空爆仓）。
2. **绝不裸空强势**：Stage1 只给候选，**涨得凶 ≠ 卖点**；必须等 Stage2 顶反转确认（顶分型+停顿K+背驰 / 创新高失败 / 放量长上影）。
3. **风险不对称**：空头利润有限、亏损不封顶 → 硬止损(顶之上)+小仓+分批，**永不扛单加仓**。
4. **OI/funding 双刃**：拥挤多头是踩踏燃料(对空头有利)，但早进就是被踩踏者且要付高费率 → 等结构先破再进。

> 这是 ①~⑤ 决策层级在「做空」方向的具体落地：Stage1 服务于②预判/③选标的，Stage2 服务于④买卖点，铁律1 强制它服从①大盘环境。
